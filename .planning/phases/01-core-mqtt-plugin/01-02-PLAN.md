    **1. src/actions/mqtt-action.ts** -- Single unified action per D-01. Extends `SingletonAction&lt;MqttActionSettings&gt;`.

    ```typescript
    import streamDeck, {
      action,
      SingletonAction,
      type KeyDownEvent,
      type WillAppearEvent,
      type WillDisappearEvent,
      type DidReceiveSettingsEvent,
    } from "@elgato/streamdeck";
    import { connectionManager } from "../services/connection-manager";
    import { topicRouter } from "../services/topic-router";
    import { brokerKey } from "../util/broker-key";
    import { logger } from "../util/logger";
    import type { GlobalSettings, MqttActionSettings, BrokerConfig } from "../types/settings";
    ```

    Use `@action({ UUID: "io.github.meintechblog.mqtt-master.mqtt" })` decorator.

    **Helper method** `getBrokerConfig(): Promise&lt;BrokerConfig | null&gt;`:
    - Call `streamDeck.settings.getGlobalSettings&lt;GlobalSettings&gt;()`
    - Return `globalSettings.broker` if it has a `host` property, otherwise return `null`
    - Log warning if no broker configured

    **onWillAppear handler** (SUB-01, SUB-02):
    - Get broker config. If null, set title to "No Broker" and return.
    - Get `brokerKey(config)` as `key`
    - Call `connectionManager.getOrCreate(config)` to ensure connection exists
    - If `settings.subscribeTopic` is non-empty:
      - Create callback: `(payload: string) => { ev.action.setTitle(payload); ev.action.setSettings({ ...settings, lastValue: payload }); }`
      - Call `topicRouter.register(key, settings.subscribeTopic, ev.action.id, callback)`
      - If register returned true (first subscriber): `connectionManager.ensureSubscribed(key, settings.subscribeTopic)`
    - If `settings.lastValue`: `ev.action.setTitle(settings.lastValue)` (restore cached value)

    **onKeyDown handler** (PUB-01, PUB-02, PUB-03):
    - Get broker config. If null, return.
    - If `settings.publishTopic` and `settings.publishPayload` are both non-empty:
      - `const client = connectionManager.getOrCreate(config)`
      - `client.publish(settings.publishTopic, settings.publishPayload, { qos: settings.qos ?? 0, retain: settings.retain ?? false })`
      - Log: `logger.info("Published to {topic}: {payload}")`

    **onWillDisappear handler**:
    - Get broker config. If null, return.
    - If `settings.subscribeTopic`:
      - `const isLast = topicRouter.unregister(key, settings.subscribeTopic, ev.action.id)`
      - If isLast: `connectionManager.ensureUnsubscribed(key, settings.subscribeTopic)`

    **onDidReceiveSettings handler**:
    - When settings change via PI, re-register subscriptions:
    - Get old settings from cached state, get new settings from event
    - If subscribeTopic changed: unregister old, register new (same logic as appear/disappear)
    - This ensures live PI changes take effect without removing/re-adding the button

    ```typescript
    import { MqttAction } from "./actions/mqtt-action";

    streamDeck.connect();
    ```

    That's it. Keep it minimal. The SDK handles the event loop.

    **1. io.github.meintechblog.mqtt-master.sdPlugin/ui/mqtt-action.html** -- Minimal PI per D-06. Plain HTML with sdpi-components. No styling (Phase 2).

    The PI needs TWO sections:
    - **Broker config** (stored in Global Settings per CONN-06): host, port
    - **Action config** (stored in Action Settings): subscribeTopic, publishTopic, publishPayload, qos, retain

    Since sdpi-components `setting` attribute auto-binds to action settings, broker config needs custom JavaScript to read/write global settings.

    Create the HTML file:
    ```html
    <!doctype html>
    <html>
    <head lang="en">
      <meta charset="utf-8" />
      <script src="https://sdpi-components.dev/releases/v3/sdpi-components.js"></script>
    </head>
    <body>
      <h2>Broker</h2>
      <sdpi-item label="Host">
        <sdpi-textfield id="broker-host" placeholder="192.168.3.8"></sdpi-textfield>
      </sdpi-item>
      <sdpi-item label="Port">
        <sdpi-textfield id="broker-port" placeholder="1883"></sdpi-textfield>
      </sdpi-item>

      <hr />
      <h2>Subscribe</h2>
      <sdpi-item label="Topic">
        <sdpi-textfield setting="subscribeTopic" placeholder="home/sensor/temperature"></sdpi-textfield>
      </sdpi-item>

      <hr />
      <h2>Publish</h2>
      <sdpi-item label="Topic">
        <sdpi-textfield setting="publishTopic" placeholder="home/light/toggle"></sdpi-textfield>
      </sdpi-item>
      <sdpi-item label="Payload">
        <sdpi-textfield setting="publishPayload" placeholder="ON"></sdpi-textfield>
      </sdpi-item>
      <sdpi-item label="QoS">
        <sdpi-select setting="qos">
          <option value="0">0 - At most once</option>
          <option value="1">1 - At least once</option>
          <option value="2">2 - Exactly once</option>
        </sdpi-select>
      </sdpi-item>
      <sdpi-item label="Retain">
        <sdpi-checkbox setting="retain"></sdpi-checkbox>
      </sdpi-item>

      <script>
        // Global settings for broker config (not auto-bound by sdpi-components)
        const hostField = document.getElementById("broker-host");
        const portField = document.getElementById("broker-port");

        // Load global settings on PI open
        SDPIComponents.streamDeckClient.getGlobalSettings().then((globalSettings) => {
          if (globalSettings.broker) {
            hostField.value = globalSettings.broker.host || "";
            portField.value = String(globalSettings.broker.port || 1883);
          }
        });

        // Save global settings on change
        function saveBrokerSettings() {
          SDPIComponents.streamDeckClient.getGlobalSettings().then((globalSettings) => {
            const broker = globalSettings.broker || {};
            broker.host = hostField.value;
            broker.port = parseInt(portField.value, 10) || 1883;
            SDPIComponents.streamDeckClient.setGlobalSettings({ ...globalSettings, broker });
          });
        }

        hostField.addEventListener("change", saveBrokerSettings);
        portField.addEventListener("change", saveBrokerSettings);
      </script>
    </body>
    </html>
    ```

    Notes:
    - The `setting` attribute on sdpi-textfield/select/checkbox auto-binds to action settings (subscribeTopic, publishTopic, etc.)
    - Broker host/port use custom JS via `SDPIComponents.streamDeckClient` to read/write global settings (per D-07, single broker)
    - sdpi-components.js loaded from CDN. If offline, executor should check if scaffold provides a local copy and use that instead.
    - No username/password fields in Phase 1 PI since test broker (192.168.3.8:1883) has no auth. The ConnectionManager already supports auth via GlobalSettings for programmatic use.
