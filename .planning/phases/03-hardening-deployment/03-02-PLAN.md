  <action>
1. Create directory `scripts/` at project root.

2. Create `scripts/deploy.sh` with the following exact content (per D-21 and D-08):
   ```bash
   #!/usr/bin/env bash
   set -euo pipefail

   REMOTE="admin@mini-von-jorg-7.local"
   PLUGIN_DIR="io.github.meintechblog.mqtt-master.sdPlugin"
   REMOTE_PLUGINS="~/Library/Application Support/com.elgato.StreamDeck/Plugins"

   echo "Building..."
   npm run build

   echo "Deploying to $REMOTE..."
   scp -r "$PLUGIN_DIR" "$REMOTE:\"$REMOTE_PLUGINS/\""

   echo "Restarting Stream Deck..."
   ssh "$REMOTE" 'pkill "Elgato Stream Deck" || true; sleep 2; open -a "Elgato Stream Deck"'

   echo "Done."
   ```
   Make the file executable: `chmod +x scripts/deploy.sh`

3. Update `package.json` scripts section to add `deploy` and `package` scripts. The full scripts object should be:
   ```json
   "scripts": {
     "build": "rollup -c",
     "watch": "rollup -c -w",
     "deploy": "bash scripts/deploy.sh",
     "package": "npm run build && streamdeck pack io.github.meintechblog.mqtt-master.sdPlugin"
   }
   ```
   Per D-22, `npm run package` builds then runs `streamdeck pack`. Note: the `streamdeck` CLI must be installed globally (`npm install -g @elgato/cli`) before `npm run package` works. This is documented in README, not enforced in the script.

4. Note on deploy.sh: The script calls `npm run build` internally, so `npm run deploy` just calls `bash scripts/deploy.sh` without chaining a separate build. This avoids double-building per research note.
