<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify end-to-end MQTT publish and subscribe on Stream Deck hardware</name>
  <what-built>Complete MQTT Master plugin: ConnectionManager, TopicRouter, unified MqttAction with publish on press and subscribe with live title updates. Deployed and running on Mac Mini with Stream Deck hardware.</what-built>
  <how-to-verify>
    **Setup (one-time):**
    1. Open Stream Deck app on Mac Mini
    2. Drag the "MQTT" action from the "MQTT" category onto a button
    3. In the Property Inspector, configure:
       - Host: `192.168.3.8`
       - Port: `1883`
       - Subscribe Topic: `test/streamdeck`
       - Publish Topic: `test/streamdeck/cmd`
       - Payload: `hello`
       - QoS: 0
       - Retain: unchecked

    **Note (ARCH-04):** Identify which Stream Deck device model you are testing on (MK.2, XL, Mini, etc.). The plugin uses `Controllers: ["Keypad"]` which should work on all models with physical buttons.

    **Test 1: Publish (PUB-01, PUB-02, PUB-03)**
    1. On Mac Mini terminal, run: `mosquitto_sub -h 192.168.3.8 -t "test/streamdeck/cmd" -v`
    2. Press the button on the Stream Deck
    3. Expected: Terminal shows `test/streamdeck/cmd hello`

    **Test 2: Subscribe (SUB-01, SUB-02)**
    1. On Mac Mini terminal, run: `mosquitto_pub -h 192.168.3.8 -t "test/streamdeck" -m "42"`
    2. Expected: Button title on Stream Deck changes to `42`
    3. Run: `mosquitto_pub -h 192.168.3.8 -t "test/streamdeck" -m "99"`
    4. Expected: Button title changes to `99`

    **Test 3: Reconnect (CONN-04)**
    1. Note the current button title
    2. Restart Mosquitto broker: `ssh root@192.168.3.8 'systemctl restart mosquitto'` (or equivalent)
    3. Wait 10 seconds for reconnect
    4. Run: `mosquitto_pub -h 192.168.3.8 -t "test/streamdeck" -m "reconnected"`
    5. Expected: Button title updates to `reconnected` (proves reconnect + resubscribe worked)

    **Test 4: Connection Sharing (CONN-05)**
    1. Add a second MQTT button with the same broker (host: 192.168.3.8, port: 1883)
    2. Configure it to subscribe to a different topic: `test/streamdeck2`
    3. Verify both buttons work: publish `mosquitto_pub -h 192.168.3.8 -t "test/streamdeck" -m "btn1"` and `mosquitto_pub -h 192.168.3.8 -t "test/streamdeck2" -m "btn2"`
    4. Expected: Both buttons update their titles independently -- two buttons with the same broker can both publish and subscribe simultaneously
    5. Optionally confirm single TCP connection: `ssh root@192.168.3.8 'mosquitto_sub -h localhost -t "$SYS/broker/clients/connected" -C 1'` or check Mosquitto log

    **Test 5: Stream Deck Restart (ARCH-01)**
    1. Restart Stream Deck app on Mac Mini
    2. Wait for it to reload
    3. Run: `mosquitto_pub -h 192.168.3.8 -t "test/streamdeck" -m "survived-restart"`
    4. Expected: Button title updates to `survived-restart`
  </how-to-verify>
  <resume-signal>Type "approved" if all 5 tests pass, or describe which tests failed and what happened. Include which Stream Deck device model was used for testing.</resume-signal>
